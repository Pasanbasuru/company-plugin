---
description: Protect state consistency, transactional integrity, and duplicate-execution safety for backend and workflow changes.
---

Use this skill when changing:
- DB writes
- workflow transitions
- background jobs
- financial/accounting logic
- retries, queues, or async flows

Check:
1. partial update risk
2. duplicate execution risk
3. ordering dependencies
4. transactional boundaries
5. idempotency
6. invariants that could be violated

Output:
- integrity risk summary
- failure modes
- safeguards
- validation strategy
