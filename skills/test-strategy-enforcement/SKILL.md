---
description: Enforce a practical testing strategy for feature work, bug fixes, and risky changes.
---

Rules:
1. New behavior should have relevant tests.
2. Bug fixes should include a regression test where practical.
3. Riskier changes need broader validation.
4. Distinguish unit, integration, e2e, and manual verification.

Output:
- recommended test layers
- required regression coverage
- minimum validation before merge
