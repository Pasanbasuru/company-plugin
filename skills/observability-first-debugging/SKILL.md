---
description: Debug production and staging issues by checking runtime evidence first before proposing changes.
---

Workflow:
1. Identify symptom and blast radius.
2. Check logs, alerts, traces, metrics, and recent deployments first.
3. Form a small set of hypotheses.
4. Prefer the smallest fix that explains the evidence.
5. State what is verified versus assumed.

Output:
- observed evidence
- likely root causes
- next debug steps
- fix recommendation
