description: Enforce frontend structure and implementation rules for React and Next.js applications without replacing UI/design plugins.
---

Use this skill for frontend work.

Guardrails:
1. Keep business logic out of presentation components.
2. Use the agreed component and folder structure.
3. Respect API/service abstraction boundaries.
4. Reuse the design system before inventing new patterns.
5. Consider accessibility, loading, empty, and error states.
6. Keep client/server boundaries explicit in Next.js.

Output:
- frontend structure check
- state/data flow concerns
- accessibility or UX risk notes
- recommended approach
